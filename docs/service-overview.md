# PLAVE Caligo Pt.2 Tracker — 서비스 소개

## 1. 서비스 개요

**PLAVE Caligo Pt.2 Tracker**는 밴드 플레이브(PLAVE)의 4번째 미니앨범 *Caligo Pt.2* 구매를 돕는 팬 메이드 웹 서비스입니다. 국내·해외에 흩어진 30여 개 판매처의 정보를 한 화면에서 비교하고, 개인의 구매·찜·드볼(전 종 수집) 진행 상황을 한곳에서 관리할 수 있게 해 줍니다.

- **대상 사용자**: 플레이브 팬덤 *별럼*(Asterum) 중에서도, 특전 미공포 포카를 최대한 모으거나 드볼(전 종)을 완성하려는 사용자
- **사용 맥락**: 앨범 예약판매 기간에 여러 판매처를 매일 비교하며 장바구니·결제·정산을 반복해야 하는 고빈도 작업
- **서비스 형태**: 모바일 우선 반응형 웹(PWA 지원), 로그인 없이 즉시 사용 가능
- **운영 주체**: 개인 팬 제작, 커뮤니티 공유 스프레드시트(음총팀 예판 정리본)를 주요 데이터 원천으로 사용

## 2. 서비스 목적

팬덤 앨범 구매 과정에는 다음과 같은 마찰이 존재합니다.

- 판매처마다 특전(미공포 포카, 포스터, 스티커 등)이 다르고, 같은 앨범이라도 버전·수량·배송비·마감일이 제각각
- 해외 판매처는 통화·환율이 다르고, 공구는 최소 수량·마감일이 별도
- 구매한 내역, 찜해둔 항목, 남은 드볼 종류를 기억하고 추적하는 일이 수동

이 서비스의 목적은 이 마찰을 제거하여 **"어디서 무엇을 어떻게 사야 가장 효율적으로 모을 수 있는가"**라는 질문에 한 번에 답하는 것입니다. 구체적으로는:

1. **가격 투명성** — 원화 기준으로 환산된 총 비용(상품가 + 배송비) 한눈에 비교
2. **특전 가시성** — 미공포 포카 이미지와 판매처를 시각적으로 매칭
3. **수집 진행도** — 드볼(전 종) 완성까지 남은 수량·종류를 실시간 계산
4. **기록 지속성** — 구매·찜 내역을 기기 간 동기화해 어디서든 이어서 관리

## 3. 핵심 기능

### 3.1 앨범 버전별 판매처 탐색
4종 버전(PHOTOBOOK / ID PASS / INVENTORY / POCAALBUM) 탭으로 해당 버전을 판매하는 판매처만 필터링해 노출합니다. 각 판매처 카드에는 가격, 통화, 특전 요약, 마감일, 현재 상태(판매중/마감/구매완료/찜)가 표시됩니다.

### 3.2 찜(장바구니) & 총액 계산
판매처를 찜에 담으면 사이드 패널의 장바구니에 추가됩니다. 수량 조절, 낱장/세트 전환이 가능하며, 무료배송 임계값을 반영한 배송비와 환율을 적용해 실시간 총액을 보여 줍니다.

### 3.3 구매 내역 관리
찜 상태의 판매처를 "구매 완료"로 전환해 수량, 결제 금액, 할인·추가 비용, 구매 일자, 메모를 기록합니다. 모든 데이터는 브라우저 localStorage에 저장되며 필요 시 삭제·수정이 가능합니다.

### 3.4 드볼(전 종 수집) 진행도
버전별로 드볼 완성에 필요한 최소 앨범 수와 가격, 병목이 되는 특전(예: 10종 포카 × 2장/앨범)을 정의해 두고, 현재 장바구니 + 구매 내역을 합산해 진행률을 실시간으로 계산합니다.

### 3.5 미공포 특전 갤러리 (별도 페이지)
모든 판매처의 미공포 포카·스티커·포스터를 그리드로 모아 보여 주는 전용 페이지입니다. 각 특전을 탭하면 모달에서 대형 이미지와 해당 판매처 정보, 마감일을 확인할 수 있고, 구매/찜 상태에 따라 테두리 색(초록=구매, 주황=찜)으로 수집 현황을 시각화합니다.

### 3.6 필터 & 정렬
- 상태 필터: 전체 / 판매중 / 찜 / 구매완료 / 마감
- 정렬 옵션: 특전 많은 순 / 마감 임박순 / 가격순
- 구매 제외 토글: 이미 구매한 판매처 숨기기
- 마감된 판매처는 자동으로 리스트 하단 정렬

### 3.7 기기 간 동기화
6자리 코드 기반 동기화(Supabase 백엔드)로 로그인 없이 PC ↔ 모바일 간 구매·찜 데이터를 공유합니다. 데이터가 변경되면 백그라운드에서 자동 동기화됩니다.

### 3.8 환율 자동 변환
KRW / USD / JPY / TWD / CNY 환율을 6시간 주기로 갱신하며, 해외 판매처 가격을 원화 기준으로 즉시 비교할 수 있게 해 줍니다. API 실패 시 기본 환율로 폴백합니다.

### 3.9 일정(캘린더) 뷰
모든 판매처·공구의 마감일을 달력 형태로 조망하고, 3일 이내 임박 건은 붉게 강조됩니다.

### 3.10 부가 기능
- 사용 가이드 모달 (첫 사용자 온보딩)
- 업데이트 로그 & 최근 저장 시각 표시
- 판매처 상세 확장 패널(오프라인 팝업스토어 주소·운영시간, 차트 반영 여부, 결제/배송 특이사항 등)
- Mixpanel 기반 사용자 행동 분석

## 5. 대표 사용자 여정

1. **앨범을 사고 싶다** — 버전 탭 선택 → 가격·특전 비교 → 판매처 카드 확장 → "구매하러 가기" → 외부 사이트 결제 → 돌아와 "구매 완료" 처리 → 드볼 진행도에 자동 반영
2. **드볼을 완성하고 싶다** — 사이드 패널에서 부족한 버전·종류 확인 → 해당 버전 탭으로 필터 → 특전 많은 순 정렬 → 가장 저렴한 조합으로 장바구니 구성
3. **특전 수집을 관리하고 싶다** — 미공포 페이지에서 그리드 탐색 → 미보유 특전을 가진 판매처 확인 → 메인으로 돌아가 장바구니에 추가
4. **기기를 옮기고 싶다** — 모바일에서 동기화 코드 생성 → PC에서 코드 입력 → 구매·찜 내역이 즉시 동기화

## 6. 디자인 관점에서의 특성

서비스의 성격이 디자인 방향에 미치는 주요 특성은 다음과 같습니다.

- **정보 밀도가 높다** — 한 화면에 판매처, 가격, 특전 이미지, 마감일, 상태 배지, 진행률 바가 공존해야 함. 정보 계층화와 공백 설계가 핵심.
- **상태가 많다** — 각 판매처는 "판매중 / 찜 / 구매완료 / 마감"의 4상태를 가지며, 필터·정렬과 조합되어 시각적 구분이 명확해야 함.
- **시각 자산이 주인공** — 미공포 포카 이미지 자체가 사용자의 구매 판단 기준이므로, 이미지 품질·비율·갤러리 레이아웃이 전체 인상을 좌우.
- **고빈도·장기 사용** — 예약판매 기간 동안 하루에도 여러 번 재방문하는 앱. 조작 비용(탭 수, 스크롤 거리)을 낮추는 것이 경험 품질과 직결.
- **모바일 우선** — 대부분의 사용자가 결제를 모바일에서 진행하므로, 터치 영역·가독성·단일 열 레이아웃을 기본으로 설계.
- **팬덤 정체성** — 플레이브 콘셉트(밤·우주·Caligo=라틴어로 '어둠')와 어울리면서도 가독성을 해치지 않는 무드가 요구됨.
